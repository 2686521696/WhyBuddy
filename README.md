## Why the name

A **slide rule** is an engineerвҖҷs analog calculator: scales, a cursor, and alignment before you trust a number.

**SlideRule** is the same idea for product decisions вҖ” not a magic вҖңone-click app factory,вҖқ but a **rehearsal instrument**:

- every step is visible
- every artifact must pass deterministic gates
- only then does a runnable application appear

If AI says вҖңdone,вҖқ that still does not count. **The gate has to pass.**

### What it is / is not

| SlideRule **is** | SlideRule **is not** |
| :--------------- | :------------------- |
| A **product rehearsal engine** (intent вҶ’ gated plan вҶ’ previewable app) | A pure **code agent** (Devin / Cursor-style repo labor) |
| A **business-structure generator** (data В· RBAC В· workflow В· pages В· AIGC) | A **chatbot / workflow builder** alone (Dify / n8n) |
| A **trust-first** system: gates, evidence, fail-closed tools | An unconstrained вҖңvibe UIвҖқ generator with no publish bar |

Closest mental models people use: *вҖңv0/Lovable for the generation surface, Power PlatformвҖ“like business structure, Manus-like long deliberation вҖ” ending in a gated app model, not a git repo.вҖқ*

## вҡҷпёҸ The V5 Rehearsal Engine

One sentence in вҶ’ multi-loop reasoning over a **capability pool** (evidence search, risk analysis, counter-arguments, synthesis, reportingвҖҰ) вҶ’ a **five-system model** (data model В· RBAC В· workflow В· pages В· AIGC) вҶ’ ships only when **publish closure** holds **6/6** evidence.

> **An AI claiming something is done does not count. Only artifacts that pass deterministic gates count.**

```mermaid
flowchart LR
  U["дёҖеҸҘиҜқж„Ҹеӣҫ<br/>One-sentence intent"] --> ORCH["Orchestrator<br/>rules + Agentic Pick"]
  ORCH --> PAR["иҪ®еҶ…е№¶иЎҢжү№<br/>parallel caps per loop<br/>(synthesis/report barriered)"]
  PAR --> GATE{"иҜҒжҚ®дҝЎд»»й—Ё<br/>structure gates В· G-GROUND"}
  GATE -->|gated_pass| STATE[("дә§зү©еә“ STATE<br/>trustLevel В· stale tracking")]
  GATE -->|fail| FEED["й”ҷиҜҜеӣһе–ӮйҮҚиҜ•<br/>error-fed retry"]
  FEED --> PAR
  STATE --> ECTX["иҜҒжҚ®дёҠдёӢж–Үз®ЎйҒ“<br/>evidence context pipeline<br/>(only gated artifacts injected)"]
  ECTX --> PAR
  STATE --> CLOSE{"еҸ‘еёғй—ӯзҺҜ<br/>publish closure 6/6"}
  CLOSE -->|closed| APP["еҸҜиҝҗиЎҢеә”з”Ё<br/>Browser Live Runtime"]
  CLOSE -->|blocked| AWAIT["AWAIT еҒңжіҠ<br/>clarify вҶ’ re-enter"]
  AWAIT --> ORCH
```

What makes it different from вҖңan LLM with a long promptвҖқ:

| Mechanism | What it does |
| :-------- | :----------- |
| **Evidence trust gate** | Every artifact passes structural + grounding gates before it earns `gated_pass`; failures re-ask with validator errors |
| **Evidence context pipeline** | Downstream reasoning is fed **only gated** upstream artifacts, priority-packed under budget with honest omission notes |
| **Publish closure** | Ships only when all six skills (dataModel В· RBAC В· workflow В· page В· AIGC В· appBundle) hold evidence вҖ” otherwise parks at AWAIT |
| **Real tools** | `web.search` and `code.run` (E2B sandbox, fail-closed without a key) via an MCP-style registry |
| **Blind-judged upgrades** | Engine changes ship with paired blind evals (A/B, position-swapped) вҖ” e.g. agentic pick 4:0, evidence pipeline 2:0 |

Deep dives: [V5.3 architecture (Chinese)](<./docs/SlideRule V5.3 жһ¶жһ„еӣҫ.md>) В· [five-system generation eval](./docs/five-system-generation-eval.md) В· [live-runtime blueprint](./docs/LIVE_SYSTEMS_BLUEPRINT.md)

## рҹҡҖ Quick Start

### Option A вҖ” Docker, one command (recommended)

Full stack (frontend + Node server + Python rehearsal engine), no local Node/Python needed вҖ” **and no database** for the main line (JSON file store):

```bash
git clone https://github.com/xiaojilele-glitch/WhyBuddy.git && cd WhyBuddy

cp .env.example .env      # fill at least LLM_API_KEY (any OpenAI-compatible provider) + SESSION_SECRET
docker compose up -d --build

# open http://localhost:3000/agent-loop/workbench
```

| Service  | Port                      | Role                                                                                          |
| :------- | :------------------------ | :-------------------------------------------------------------------------------------------- |
| `app`    | `3000` (host) вҶ’ `3001`    | Node server + bundled frontend; SlideRule API thin-proxies to Python                          |
| `python` | `9700` (network-internal) | V5 rehearsal engine: five-system generation, evidence trust gates, evidence pipeline, closure |

`mysql` is an **optional profile**, only for legacy accounts (login / email codes / projects): `docker compose --profile accounts up -d`.

Sessions and artifacts persist in the named volume `sliderule-python-data` вҖ” rebuilds keep your data.

```bash
docker compose logs -f app python   # follow logs
docker compose up -d --build        # rebuild after pulling updates
docker compose down                 # stop (keeps data volumes)
docker compose down -v              # stop and wipe data
```

<details>
<summary>рҹ“Ң <strong>Deployment notes</strong></summary>

- **Required env**: `LLM_API_KEY` / `LLM_BASE_URL` / `LLM_MODEL` (any OpenAI-compatible provider) and `SESSION_SECRET` (use a 64-char random hex in production). Without an LLM key the stack still boots; rehearsals fall back to deterministic templates.
- **Optional**: `WEB_SEARCH_API_KEY` (grounded web evidence) and `E2B_API_KEY` (sandboxed `code.run`) вҖ” missing keys fail closed; tools stay unavailable.
- **Port conflicts**: change `app`вҖҷs `ports` mapping in `docker-compose.yml` (e.g. `"8080:3001"`).
- **Accounts (optional)**: the rehearsal main line needs no database. Enable accounts with `docker compose --profile accounts up -d`.
- **Production servers вҖ” pull, donвҖҷt build**: releases to `main` build images to ghcr.io (`.github/workflows/deploy-images.yml`). On the server:

  ```bash
  docker compose -f docker-compose.prod.yml pull && docker compose -f docker-compose.prod.yml up -d
  # auto-updates (Watchtower every 5 min):
  docker compose -f docker-compose.prod.yml --profile auto up -d
  # rollback: pin :latest to a release :<commit-sha> in docker-compose.prod.yml
  # slow ghcr (e.g. China): SLIDERULE_REGISTRY=ghcr.nju.edu.cn in .env
  # or Docker Hub dual-push (with secrets configured):
  #   SLIDERULE_IMAGE_APP=docker.io/<hub-user>/whybuddy-app:latest
  #   SLIDERULE_IMAGE_PYTHON=docker.io/<hub-user>/whybuddy-python:latest
  ```

- **Corporate TLS-intercepting proxies**: drop your root CA (`.crt` PEM) into `docker/certs/` before building (see `docker/certs/README.md`). Certificates are gitignored.
- **Not in compose**: Lobster Executor (DinD, opt-in), Redis (off by default), Feishu (mock by default).
- `.env` is never baked into images; it is injected at runtime via `env_file`.

</details>

## вҡ”пёҸ How to place SlideRule

These tools solve **different jobs**. The table is not вҖңwe replace them allвҖқ вҖ” it shows where the rehearsal main line is unique.

| Capability | Agent frameworks<br/>(CrewAI / LangGraph) | Workflow builders<br/>(Dify / n8n) | **SlideRule** |
| :--------- | :----------------------------------------: | :--------------------------------: | :-----------: |
| Open source | вң… | вң… | вң… |
| Multi-agent / long orchestration | вң… | вҡ пёҸ | вң… |
| One sentence вҶ’ **product structure** (data В· RBAC В· flow В· pages) | вқҢ | вқҢ | вң… |
| Spec package (requirements В· design В· tasks В· traceability) | вқҢ | вқҢ | вң… |
| **Evidence-gated publish closure** | вқҢ | вқҢ | вң… |
| Rehearsed model **runs as an app** in the browser | вқҢ | вқҢ | вң… |
| Replay, audit, human park / re-enter | вҡ пёҸ | вҡ пёҸ | вң… |
| Sandboxed code tools | вҡ пёҸ | вҡ пёҸ | вң… |
| Browser-only demo (zero install) | вқҢ | вқҢ | вң… |

For generation UX people often compare **v0 / Lovable / Bolt**; for long visible deliberation, **Manus-class agents**; for enterprise app structure, **Power Platform / low-code**. SlideRuleвҖҷs bet is the **intersection**: *rehearse the business system under gates, then run the model вҖ” not only emit code or a chat bot.*

## рҹӨқ Contributing

```bash
1. Fork & clone вҶ’ pnpm install
2. pnpm run dev:frontend (UI) or pnpm run dev:all (full stack)
```

**Branch model**: `main` is production; `pre_main` is daily integration. Merges go through the release gate вҖ” a red gate mechanically blocks the merge:

```bash

bash scripts/merge-gated.sh pre_main "<release message>" main    # release вҶ’ main
```
