# 逆向评测：SlideRule 一句话推演 vs 真实系统架构（AI 美工 SaaS）

> 正向 vs 逆向的对照实验：`docs/系统 Mermaid 架构图.md` 是从一个**真实运营中的系统**
> （img.xiaowenan.com，ThinkPHP 多应用 + Nuxt3/Vue3 多租户 AI 创作 SaaS）逆向提炼的
> as-built 架构（ground truth）；本报告把 SlideRule 从一句话意图**正向推演**出的五系统
> 模型与它逐项比对，诚实计算覆盖率、列出缺口。

### 页面

## 五、不变式层：0/8 —— 模板级缺口的实证

Ground truth 第七节的 8 条系统不变式（支付只认服务端回调、扣费先于上游请求、媒体结果
必须转存脱离上游临时 URL、任务-扣费-余额同链路可追溯……）在 SlideRule 交付物中**没有
对应表达层**。个别不变式有间接痕迹（生成任务.消耗次数 + 次数钱包 ≈ 可追溯的数据基础），
但作为陈述性约束是完全缺失的。这正是产品模型里最能证明"想清楚了"的一层。

## 六、结论

### 覆盖率汇总

| 维度 | 覆盖率 | 幻觉/自创 |
| --- | --- | --- |
| 数据模型 | **70%**（14/20 域实体） | 0 幻觉实体 |
| RBAC 端面 | **100%**（3/3 端 + 细粒度权限） | 无 |
| 页面 | **63%**（10/16） | 1 页随自创流带出 |
| AIGC | 点名能力 **3/3**；全量 3/8 | 2 项合理增强 |
| Workflow 链路 | **0/5 完整**，1 条部分 | 1 条自创审核流 |
| 系统不变式 | **0/8** | —（无此表达层） |

### 一句话读法

一句话意图换来了真实系统**静态结构面约七成**的还原（数据+权限+页面），且零幻觉、
引用完整（门禁 0 findings）；**动态行为面（业务链路）和约束面（不变式）是两个成体系的
缺口**——前者是生成器只产一条审批流的模板局限，后者是交付物 schema 里根本没有这一层。

### 对 SlideRule 的三条改进方向（按杠杆排序）

1. **Workflow 从"单条审批流"扩展为"多条业务链路"**：至少覆盖核心对象生命周期
   （任务：创建→计费→执行→归档）与资金链路（下单→回调→改账），这是覆盖率损失最大的一块。
2. **交付物增加"系统不变式"层**：每个域 5–8 条陈述性约束（谁先谁后、什么只认什么、
   什么必须落库）。本实验证明成熟系统的架构文档必然沉淀这一层。
3. **意图粒度实验**：数字人/语音/抠图等缺失能力全部是意图句未点名的——值得验证
   "更完整的一句话"能否线性换回召回率，还是存在生成器上限。

### 公平性与诚实性注记

- 意图句只点名了图片/视频/对话三项 AIGC 能力，缺失的 5 项不应全记为模型失误；
  点名项命中率是 3/3。
- 登录闸/演示闸/中间件链属工程实现层，产品模型不表达它们是合理边界，未计入分母
  （若计入，RBAC 覆盖率降为 4/6）。
- "租户配置审核流"与"提示词优化"等自创项在真实系统中不存在，本报告如实标注为增强而
  非命中，不参与覆盖率分子。
- 比对为人工映射（非 LLM 打分），生成名与快照均入库可复核。
