# SlideRule 北极星（North Star）

> 本文档是项目的唯一方向文件。任何新功能、新方向、新重构，先回答一个问题：
> **它是否让"一句话 → 应用方案"这条路径更短、更稳、更可信？** 不是，就不做。

## 一句话定义

**输入一句意图，喝杯咖啡的工夫，得到一个可预览、可导出、带证据链的应用方案。**

> 我们不承诺秒回。LLM 的深思需要时间——SlideRule 承诺的是**推演全程可见、
> 每一步有证据、每道门真实执行**，以及不浪费你任何一秒在工程内耗上
> （能并行的能力绝不串行排队）。

- **可预览**：生成的方案能在页面上直接看到效果（系统屏 / 图表 / 页面预览），不是一堆纯文本。
- **可导出**：规格文档、架构图、提示词包可以完整带走。
- **带证据链**：每个结论有出处，每个 gate 由脚本实际执行并留痕，而不是模型口头声称。

## 黄金路径（Golden Path）

主线的验收标准是下面这条端到端路径**流畅、稳定、有 smoke test 守护**：

```
一句话意图
  → 推演出规格（spec）
  → 3 个 Skill 联动生成（数据中台 → 权限 RBAC → 页面设计器）
  → 系统屏预览
  → 一键导出（spec 包 + 证据链）
```

- 六个 Skill（数据中台、权限、工作流、页面设计器、应用中心、AIGC 中台）中，
  **先做深上面 3 个**，其余 3 个保持骨架，等黄金路径打磨完再扩。
- Skill 的运行时逻辑**只存在于 Python 侧**（`slide-rule-python/`）；
  `client/src/lib/skills/` 的 TS 内核只承担类型契约与校验，不再实现运行时。

## 后端策略

- **Python（`slide-rule-python/`，FastAPI）是唯一后端演进方向。**
- Node（`server/`）冻结为薄代理：不加新功能，只保留委托与兼容层，
  legacy 分支（`SLIDERULE_V5_BACKEND=legacy` 才加载的代码）逐步删除。
- 迁移进度与取舍见 [`docs/NODE_PYTHON_PARITY.md`](./NODE_PYTHON_PARITY.md)。
- **`verify:sliderule-v5` 是唯一的发布门**：过了才能合入，AI 声称完成不算数，脚本跑过才算数。

## 判断一个功能做不做（决策清单）

1. 它在黄金路径上吗？不在 → 记入 backlog，不做。
2. 它需要动 Node 后端吗？需要 → 改成在 Python 侧实现，或不做。
3. 它服务于 6 个 Skill 之外的新概念吗？是 → 不做，先收敛。
4. 它做完后 `verify:sliderule-v5` 能覆盖到吗？不能 → 先补 gate 再做功能。
