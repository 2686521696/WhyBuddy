# 发布全流程

从一句话到一个能操作的应用，SlideRule 的完整链路是这样走的。每一步都有
确定性检查兜底——**AI 声称完成不算数，过了门的产物才算数**。

## 第 1 步 · 说出你的想法

在首页输入框描述业务（或点模式卡/快速开始填入模板），也可以：

- **上传需求文档**（txt/md/docx 文本类直接注入；PDF 走沙盒提取）
- **上传页面截图**（视觉 AI 自动提取图里的文字与界面结构，注入推演指令）
- 点 ✨ **优化提示词**，把一句话意图改写成信息更全的推演提示词

## 第 2 步 · 实时推演

发送后左栏直播推演过程：理解意图 → 检索外部证据 → 澄清需求 → 分析风险 →
起草五系统模型 → 多轮收敛。由智能编排决定每轮做什么（详见术语表
「智能编排」），期间引擎可能停下来**问你问题**——回答越具体，应用越贴合。

中途刷新页面、断网、换设备都不怕：推演在服务端继续跑，回来自动接回直播
（断线重生）。

## 第 3 步 · 过确定性门

推演产出的五系统模型要过两道机械检查：

- **结构门**：六段模型必须齐全，且所有跨系统引用（流程节点的角色、页面
  绑定的字段等）必须真实存在——引用悬空直接拦下，不放行
- **覆盖门**：六个系统的证据必须齐备，缺一段就是被拦（blocked）

**过门 → 右上角出现 `closed 6/6` 徽标，右侧应用接管舞台。**
**被拦 → 显示「发布检查未通过」+ 缺什么，点「补齐缺口」只重跑缺的部分。**

## 第 4 步 · 操作你的应用

右侧就是能用的应用：侧边导航切页面、右上角切角色（不同角色看到的页面和
操作不同，这就是 RBAC 在起作用）、直接录入真实数据（存在本机浏览器）。
顶部可切换 **桌面 / 手机 / 代码** 三种视图。

## 第 5 步 · 不满意就迭代

- **补充指令**：直接再发一句（如"角色里加一个志愿者"），引擎在现有应用
  基础上**增量精修**，无关部分逐字节保留，不会推倒重来
- **版本前进/回退**：每次模型变化自动存档，工具栏 ◀ ▶ 一键回到任意版本
- **挑战结论**：对某轮结论点「挑战」，引擎针对性重推该部分

## 第 6 步 · 带走成果

- **交付物**：右上角「交付物」面板——可行性报告、任务清单、追溯矩阵等，
  一键导出 Markdown
- **代码**：代码视图里「打包导出」，整个应用工程打成 zip
- **部署**：仓库自带 Docker 一键部署（`docker compose up -d`），详见
  「环境与配置」

## 附 · 发版是怎么保真的（工程侧）

本产品自己的每次更新也走同一哲学：改动必须通过全量测试 + 真实浏览器冒烟
（发布门），全绿才允许合并进生产分支。你在「更新日志」里看到的每一条，
都是过了门的事实，不是计划。
