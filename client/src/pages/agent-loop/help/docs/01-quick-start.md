# 快速上手

SlideRule 把一句话业务需求，推演成一个**可以直接操作的应用**——不是原型图，是带数据模型、权限、流程、页面和 AI 能力的运行系统。

## 三步生成你的第一个应用

### 1. 创建应用

在工作台点右上角 **「+ 创建新应用」**，或在左侧栏点「新建会话」。

### 2. 用一句话说清你要什么

在输入框里描述你的业务，比如：

> 社区宠物医院预约问诊系统——预约、分诊、复诊提醒一体化

不用写需求文档。说人话就行，推演引擎会追问它需要知道的事。

**一个好话题的要素**（可选，说得越清楚推演越准）：

- 给谁用：目标用户是谁（商家 / 个人 / 团队）
- 管什么：核心对象是什么（预约单、会员卡、工单……）
- 走什么流程：从发起到结束大概几步

### 3. 等推演闭环

发送后左栏会实时直播推演过程：理解意图 → 检索外部证据 → 澄清需求 → 分析风险 → 起草五系统模型 → 多轮推演收敛。

期间引擎可能**停下来问你问题**（目标用户是谁、范围多大）——回答得越具体，生成的应用越贴合。

当右上角出现 **closed 6/6** 徽标、右侧应用接管舞台时，你的应用就生成好了。点侧边导航可以切换页面，右上角可以切换角色，直接开始录入真实数据。

## 生成之后能做什么

| 操作 | 位置 | 说明 |
|---|---|---|
| 操作应用 | 右侧舞台 | 真实增删改查，数据存在本机浏览器 |
| 切换角色 | 应用右上角 | 不同角色看到的页面和按钮不同（权限系统生效） |
| 追问 / 修改 | 底部输入框 | 继续对话，让引擎调整模型 |
| 挑战结论 | 结论卡片 | 不同意某个设计？点「挑战此结论」让引擎重新推演 |
| 游标透视 | 舞台右上「游标」 | 悬停任意元素，看它背后的声明来自哪条模型 |
