# 推演页指南

推演页是「左栏对话直播 + 右侧应用舞台」的双栏结构。

## 左栏：想法直播

推演进行中，左栏按阶段实时流出引擎的每一步：

- **步骤条**：理解意图 → 第 N 轮推演（检索证据 / 澄清需求 / 生成路线 / 分析风险）→ 五系统模型起草 → 发布闭环。进行中的阶段带蓝色脉冲点，完成的打绿勾。
- **实时正文**：LLM 分析的原文逐字流出；五系统 JSON 起草时显示尾窗和字符数。
- **完成后折叠**为「推演过程 · N 阶段 · M 步」，点击可展开回看。**刷新页面不丢**——时间线随会话持久化，回来还能完整回放。

## 右侧：应用舞台

- **推演中**：极简「推演中 + 当前动作」占位（刻意不重复左栏直播）。
- **闭环后**：应用接管舞台，顶条显示话题 + `运行中` 徽标。

### 舞台顶条控件

| 控件 | 作用 |
|---|---|
| 桌面 / 手机 / 代码 | 切换预览档位：响应式布局 / 移动端视图 / 查看生成的模型源码 |
| 游标 | 开启 X 光透视（见下） |

### 游标透视

计算尺的游标对准刻度读数——SlideRule 的游标对准界面元素，读出它**背后的声明**：

1. 点舞台右上「游标」开启
2. 悬停应用里的任意字段、按钮、页面
3. 右侧面板显示：这个元素绑定哪个实体字段、哪条权限声明让它对当前角色可见、参与哪条工作流

这是「模型是唯一事实源」的可视化证明：界面上没有任何元素是凭空画的。

## 对话操作

| 操作 | 方式 |
|---|---|
| 追问 / 补充需求 | 直接输入发送，引擎增量推演 |
| 回答引擎的问题 | 引擎停泊等待时，答案会解锁对应缺口 |
| 挑战结论 | 结论卡片上点「挑战此结论」，该结论标记为过期并重新推演 |
| 中断推演 | 发送按钮在推演中变为停止方块 |
| 重置会话 | 右上角「重置会话」清空当前话题重新开始 |
