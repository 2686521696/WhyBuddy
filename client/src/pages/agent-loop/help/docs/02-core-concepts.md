# 核心概念

理解四个概念，就理解了 SlideRule 和「AI 画原型图」工具的本质区别。

## 五系统模型：应用的唯一事实源

每个应用背后是一份结构化的**五系统模型**，而不是一堆画布上的矩形：

| 系统 | 声明什么 | 例子 |
|---|---|---|
| 数据模型 datamodel | 实体和字段 | 患者（姓名/手机号/病史）、预约单（时间/医生/状态） |
| 权限 rbac | 角色 · 权限 · 菜单 | 前台能建预约，医生能写病历，财务只看账单 |
| 工作流 workflow | 节点和流转 | 提交预约 → 前台确认 → 分诊 → 就诊 → 回访 |
| 页面 page | 页面 · 字段绑定 · 按钮权限 | 预约日历页绑定预约单实体，「取消」按钮仅前台可见 |
| AI 能力 aigc | 应用内的 AI 功能 | 问诊录音转病历草稿、复诊提醒文案生成 |

你看到的每个页面、每个按钮、每条数据规则，都是从这份模型**推演渲染**出来的——模型是源，界面是像。这就是为什么预览不能手工拖拽修改：改了像不改源，系统就撒谎了。要改，就通过对话改模型，界面自然跟着变。

## 证据 n/6：闭环信任门

「AI 声称完成不算数，脚本跑过才算数」是这个系统的北极星。应用生成不靠 AI 自己说"我做完了"，而是要过**发布闭环门**：六个系统维度各自交出证据，全齐（6/6）且无阻塞，应用才标记为可运行。

工作台卡片右下角的 **6/6 绿字**就是这个含义——它不是质量评分，是证据清点。差几项就是差几项，系统不会圆谎。

## 推演回合：对话即建模

每次你发消息，引擎跑一个**推演回合**：规划能力路线 → 逐个执行（检索证据 / 澄清 / 分析风险 / 起草模型）→ 信任门验收 → 覆盖度检查。左栏的"想法直播"就是这个过程的实时叙述，刷新页面后仍可回放。

引擎不确定的事**会停下来问你**（停泊），而不是编一个答案继续跑。

## 诚实降级

任何环节失败——LLM 网关超时、搜索源全挂、模型解析不过门——系统显示真实状态（降级 / 待补充 / 未闭环），不用假数据补位。看到「推演未闭环」就是真的没闭环，看到证据 2/6 就是真的只有两项证据。
