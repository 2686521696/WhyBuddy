# 常见问题

## 为什么我的应用显示「推演中」而不是「可运行」？

可运行需要同时满足：闭环证据 6/6、无阻塞、**且五系统模型完整落库**。常见原因：

- 推演还没跑完 → 等它收敛，或进入会话看它停在哪
- 引擎在等你回答问题（状态是「待补充」）→ 进会话回答
- 证据齐了但模型没落库（早期版本的会话）→ 重新驱动一轮即可

## 刷新页面后推演过程还在吗？

在。左栏时间线随会话持久化（保留最近 3 轮），刷新后完整回放。更早版本生成的会话没有这份数据，会显示精简骨架——重新跑一轮后就有了。

## 应用里的数据存在哪？会丢吗？

你在生成应用里录入的数据存在**本机浏览器**（localStorage），不上传服务器。换浏览器/清缓存会丢。它定位是排练数据——验证应用逻辑用，不是生产库。

## 为什么预览不能直接拖拽改界面？

因为界面是从五系统模型推演渲染的**像**，模型才是**源**。允许手改预览等于允许像和源不一致——那就回到了"原型图好看但和实现两张皮"的老路。要改界面，用对话改模型，界面自动跟上。

## 工作台里一堆测试会话怎么清理？

悬停卡片右上「…」→ 删除应用。开发时跑测试产生的垃圾会话（新版已隔离，不会再产生）都可以这样删。

## 推演特别慢或者失败了怎么办？

1. 点工作台的**服务健康点**，展开看三服务哪个红了
2. LLM 通道红 → 检查 `.env` 里的 `LLM_API_KEY` / `LLM_BASE_URL`，重启 Python 后端
3. Python 引擎红 → 确认 `uvicorn` 在 9700 端口跑着
4. 都绿但推演中断 → 左栏会显示降级原因，直接把报错发给开发者

## 生成的应用能导出部署吗？

当前版本应用运行在 SlideRule 运行时里（模型驱动渲染）。独立导出部署在路线图上，还没有交付时间。
