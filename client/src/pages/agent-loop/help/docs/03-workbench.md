# 应用中心指南

应用中心是所有推演产物的画廊，分两个 tab：**我的应用**（你的会话生成的应用）与**官方示例库**（过了发布门的演示应用，点卡即以该场景起手推演）。

## 卡片读法（两个 tab 同一套卡片）

卡片一律 16:9，画面铺满整卡，字段内容压在底部浮层上：

```
┌──────────────────────────────┐
│                              │
│        [16:9 画面]           │  ← 我的应用：按真实模型渲染的迷你应用
│                              │     官方示例：实跑闭环后的真实截图
│ ▩ 采买通                     │  ← 品牌图标（身份主题色）+ 产品名
│ 页面 4 角色 4 AI 3  ●closed 6/6│  ← 真实指标 + 门语言状态
└──────────────────────────────┘
```

- **画面不是摆拍**：我的应用的缩略图由持久化五系统模型现场渲染（深色导航里是真实页面名）；官方示例是 playwright 实跑闭环后拍摄的真截图。未闭环的应用显示推演态和证据进度条，不摆假图。
- **状态 = 门语言**：`closed 6/6`（发布闭环全过，可打开操作）、`推演中`（还在收敛）、`blocked`（引擎停下来等你补充信息）。6/6 不是质量评分，是证据清点。

## 工具行

| 控件 | 作用 |
|---|---|
| 搜索框 | 我的应用按话题过滤；示例库按产品名/意图/分类过滤 |
| 全部 / 推演中 / closed 6/6 / blocked | 我的应用 tab 的门语言筛选卡（含各状态数量） |
| 分类 chips | 示例库 tab 按场景分类（供应链 / 人力资源 / 客户服务…） |
| 服务健康点 | **点击展开**三服务详情：Node API / Python 推演引擎 / LLM 通道 |
| 最近更新 | 我的应用 tab 点击切换排序方向（最近 ↔ 最早） |

## 分页

一页最多 12 张（4 × 3）；超出走底部分页器。切换筛选/搜索/tab 自动回到第一页。

## 卡片操作

- **点我的应用卡片** → 激活该会话并进入推演页，应用直接可操作
- **点官方示例卡片** → 新建会话并预填该场景的起手意图，回车即开始推演（走同一条推演管线，不是复制死模板）
- **悬停右上「…」→ 删除应用**（仅我的应用）→ 删除该会话（幂等操作，误删的测试会话可以这样清理）
