# 需求文档：获取设备信息节点

## 目标

定义 `get_device_info` 节点获取设备环境信息的能力，并在缺少宿主 SDK 的情况下提供服务端安全降级实现。

## 需求

### 需求 1：设备信息

系统应支持获取终端设备、浏览器或运行环境的基本信息。
当前最小闭环至少应支持基于请求头、Client Hints 与宿主显式 hint 输出结构化摘要。

### 需求 2：权限边界

获取设备信息应明确用户授权与隐私边界。
系统不得回传原始 `user-agent`、IP、Cookie、Authorization 或高指纹字段，而应仅返回过滤后的摘要信息。

### 需求 3：可选迁移

该节点应视宿主产品场景决定是否真正落地。
在 Cube 当前阶段允许采用“安全降级版”最小实现，后续再由宿主前端补充更丰富的设备上下文。
