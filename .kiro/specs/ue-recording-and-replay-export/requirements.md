# 需求文档：录制与回放导出

## 简介

本规格定义 UE5 实时演出的录制、回放与导出功能。系统需支持将演出过程录制为视频 clip，与现有 replay 系统对接，并支持导出为短视频、GIF 和截图。

## 需求

### 需求 1：实时录制

**用户故事：** 作为用户，我希望能录制 UE 场景的演出过程，以便后续回看和分享。

#### 验收标准

1. 系统应支持一键开始和停止录制。
2. 录制应同时保存视频流和指令时间线。
3. 录制过程不应明显影响渲染帧率（帧率下降 ≤ 10%）。

### 需求 2：回放

**用户故事：** 作为用户，我希望能回放之前的演出，以便复盘任务执行过程。

#### 验收标准

1. 系统应支持基于指令时间线的精确回放。
2. 回放应支持速度控制（0.5x / 1x / 2x）。
3. 回放应支持进度条拖拽跳转。

### 需求 3：导出

**用户故事：** 作为用户，我希望将演出导出为视频或图片，以便在其他平台分享。

#### 验收标准

1. 系统应支持导出为 MP4 视频（H.264 编码）。
2. 系统应支持导出为 GIF 动图。
3. 系统应支持导出当前画面为 PNG 截图。
4. 导出应支持选择分辨率和画质。

### 需求 4：与 replay 系统对接

**用户故事：** 作为系统，我希望 UE 录制数据能与现有 replay 系统集成，以便统一管理。

#### 验收标准

1. 录制数据应存储在现有 replay 目录结构中。
2. replay 页面应能展示 UE 录制的回放。
3. 录制元数据应关联到对应的 mission。
