# 需求文档：本地资源与会话治理

## 简介

本规格定义本地 GPU 资源监控、显存告警、空闲自动暂停与联合成本面板。系统需在本地运行 UE5 时持续监控资源占用，在资源紧张时主动降级或暂停，并提供 LLM token 与本地渲染的联合成本视图。

## 需求

### 需求 1：GPU 资源监控

**用户故事：** 作为用户，我希望了解 UE5 对本地 GPU 的占用情况，以便判断是否需要调整画质。

#### 验收标准

1. 系统应实时监控 GPU 占用率、显存使用量、GPU 温度。
2. 监控数据应以 1 秒间隔更新。
3. 监控数据应在前端面板中可视化展示。

### 需求 2：显存阈值告警

**用户故事：** 作为系统，我希望在显存即将耗尽时发出告警，以便提前采取措施避免崩溃。

#### 验收标准

1. 系统应在显存使用超过 80% 时发出黄色告警。
2. 系统应在显存使用超过 90% 时发出红色告警并建议降低画质。
3. 系统应在显存使用超过 95% 时自动降低画质档位。

### 需求 3：空闲自动暂停

**用户故事：** 作为用户，我希望在长时间不操作时 UE 自动暂停渲染，以便节省电力和 GPU 资源。

#### 验收标准

1. 系统应在用户无操作 5 分钟后自动暂停 UE 渲染。
2. 暂停时应显示最后一帧的静态截图。
3. 用户操作时应在 2 秒内恢复渲染。
4. 暂停时间阈值应可配置。

### 需求 4：联合成本面板

**用户故事：** 作为用户，我希望看到 LLM 调用和本地渲染的综合成本，以便了解资源消耗。

#### 验收标准

1. 面板应展示 LLM token 消耗与估算费用。
2. 面板应展示本地 GPU 运行时长与估算电力成本。
3. 面板应展示当前会话的累计成本。
