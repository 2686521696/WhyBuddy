## 2026-04-23 中文收口说明

本轮收口聚焦的是当前主仓已经落地的最小可观测与审计事实，不额外扩写尚未最终锁版的治理结果。

当前可按事实确认的内容如下：

- 已形成最小事件目录，主线已覆盖节点开始、完成、失败、变量赋值、边跳转、循环、等待输入、重试、升级、终止等关键事件语义。
- `replay / audit` 已具备最小镜像闭环，当前主仓可将关键 runtime 事件映射为审计侧可消费的记录。
- 事件目录已经足以支撑当前 Web-AIGC 主线的最小观测面、回放面与审计面协同。

当前需要保留的真实边界如下：

- 可以确认“最小 event catalog + replay / audit 闭环成立”。
- 不能扩写为“所有 runtime 事件都已完成全量 lineage 统一沉淀”。
- `lineage` 更准确地说仍属于后续主线持续收口项，需要在最终锁版时继续核对口径。

因此，本 spec 的任务可按完成处理，但结论应限定为：当前主仓已经形成最小可观测与审计闭环，而不是一次性完成全部治理归并。
