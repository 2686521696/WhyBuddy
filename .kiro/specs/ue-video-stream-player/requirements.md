# 需求文档：前端视频流播放器

## 简介

本规格定义前端视频流播放器组件，用于替换当前 Three.js Canvas 为 UE5 Pixel Streaming 视频流。播放器需支持本地 WebRTC 流接入、连接状态管理、自适应画质，以及 UE 不可用时回退到 Three.js 或预渲染视频的降级策略。

## 需求

### 需求 1：WebRTC 视频流接入与播放

**用户故事：** 作为用户，我希望在浏览器中看到 UE5 实时渲染的 3D 场景画面，以便获得高品质视觉体验。

#### 验收标准

1. 系统应提供 `<VideoStreamPlayer>` React 组件，接收信令服务地址并建立 WebRTC 连接。
2. 视频流应以 `<video>` 元素渲染，支持自适应容器尺寸。
3. 组件应在连接建立后 3 秒内开始显示画面。
4. 组件应暴露 `onConnected`、`onDisconnected`、`onError` 回调。

### 需求 2：连接生命周期管理

**用户故事：** 作为用户，我希望视频流在网络波动时能自动恢复，以便不中断观看体验。

#### 验收标准

1. 当 WebRTC 连接断开时，组件应自动发起重连，最多重试 3 次。
2. 重连期间应显示"正在重新连接"的加载状态。
3. 若重连失败，组件应触发降级回调，通知父组件切换到备用渲染模式。
4. 组件卸载时应主动关闭 WebRTC 连接并释放媒体资源。

### 需求 3：降级回退策略

**用户故事：** 作为用户，我希望在 UE 渲染不可用时仍能看到 3D 场景，以便核心功能不受影响。

#### 验收标准

1. 当 UE 实例未启动或连接失败时，系统应自动回退到 Three.js Canvas 渲染。
2. 当网络带宽不足时，系统应降低视频流分辨率或帧率。
3. 降级状态应在 UI 上有明确提示（如"当前为简化渲染模式"）。
4. 从降级模式恢复到 UE 流模式时，切换过程应平滑无闪烁。

### 需求 4：画质与性能自适应

**用户故事：** 作为用户，我希望视频流画质能根据我的设备性能自动调整，以便获得流畅体验。

#### 验收标准

1. 系统应根据当前帧率和网络延迟动态调整请求的视频分辨率。
2. 系统应支持手动切换画质档位（高 / 中 / 低）。
3. 画质切换过程中不应出现画面中断。
