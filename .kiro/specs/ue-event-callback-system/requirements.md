# 需求文档：事件回调系统

## 简介

本规格定义 UE5 向前端发送事件回调的机制。UE 场景中的关键事件（动画完成、镜头就位、角色到达、场景加载完成等）需要通知前端，用于 UI 时序对齐（如等动画播完再切下一步）。系统需保证事件的可靠投递、格式统一与超时处理。

## 需求

### 需求 1：事件类型定义

**用户故事：** 作为前端应用，我希望收到 UE 场景中关键事件的通知，以便在正确的时机更新 UI。

#### 验收标准

1. 系统应支持以下事件类型：animation.completed、camera.arrived、character.reached、scene.loaded、effect.finished。
2. 每个事件应包含事件类型、时间戳、关联实体 ID 和附加数据。
3. 事件应通过 WebSocket 实时推送到前端。
4. 事件格式应遵循统一的 JSON schema。

### 需求 2：事件可靠性保证

**用户故事：** 作为系统，我希望关键事件不会丢失，以便前端时序逻辑不会因事件丢失而卡死。

#### 验收标准

1. 系统应为每个事件分配唯一 eventId。
2. 系统应支持事件确认机制（ACK），前端收到事件后应回复确认。
3. 未确认的事件应在 3 秒后重发，最多重发 2 次。
4. 系统应维护事件日志，支持查询最近 100 条事件。

### 需求 3：超时处理

**用户故事：** 作为前端应用，我希望在等待 UE 事件超时时能收到通知，以便进行降级处理。

#### 验收标准

1. 前端应支持为每个等待的事件设置超时时间（默认 10 秒）。
2. 超时后应触发 timeout 回调，并附带原始等待条件。
3. 系统应支持取消正在等待的事件。
