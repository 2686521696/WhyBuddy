# 需求文档：镜头系统

## 简介

本规格定义 UE5 中的镜头机位系统，包括机位预设、镜头切换过渡动画与自动构图规则。镜头系统由 LLM 导演通过指令协议驱动，支持全景、特写、跟随、俯瞰、会议室等多种机位，并根据活跃角色数量和位置自动调整构图。

## 需求

### 需求 1：镜头机位预设

**用户故事：** 作为导演系统，我希望能快速切换到预定义的镜头机位，以便高效编排分镜。

#### 验收标准

1. 系统应提供以下预设机位：全景（overview）、特写（closeup）、跟随（follow）、俯瞰（topdown）、会议室（meeting）。
2. 每个预设应定义位置、旋转、FOV 和焦距参数。
3. 系统应支持通过指令协议切换预设机位。
4. 预设参数应支持通过配置文件自定义。

### 需求 2：镜头过渡动画

**用户故事：** 作为用户，我希望镜头切换时有平滑的过渡效果，以便获得电影级的观看体验。

#### 验收标准

1. 系统应支持以下过渡类型：淡入淡出（fade）、推拉（dolly）、环绕（orbit）、直切（cut）。
2. 过渡动画时长应可配置（默认 1.0 秒）。
3. 过渡期间不应出现画面撕裂或跳帧。
4. 系统应支持过渡动画的缓动曲线配置（ease-in / ease-out / ease-in-out）。

### 需求 3：自动构图

**用户故事：** 作为导演系统，我希望镜头能根据活跃角色自动调整构图，以便减少手动调参。

#### 验收标准

1. 当只有 1 个活跃角色时，系统应自动切换到特写或中景。
2. 当有 2-3 个活跃角色时，系统应自动调整 FOV 使所有角色在画面内。
3. 当有 4+ 个活跃角色时，系统应切换到全景机位。
4. 自动构图应遵循三分法则，角色不应出现在画面边缘。

### 需求 4：镜头跟随

**用户故事：** 作为用户，我希望镜头能跟随正在行动的角色，以便不错过关键动作。

#### 验收标准

1. 系统应支持锁定跟随指定角色，镜头平滑跟随角色移动。
2. 跟随模式下应保持角色在画面中心偏左或偏右（三分法）。
3. 角色停止移动后，镜头应在 2 秒后自动解除跟随。
4. 跟随过程中应避免穿墙或穿过家具。
