# 需求文档：降级与兜底策略

## 简介

本规格定义 UE5 渲染不可用时的降级与兜底策略。系统需在 UE 不可用时回退到 Three.js，在低带宽时降低分辨率，在移动端走预渲染视频，并在降级状态下提供明确的 UI 提示。

## 需求

### 需求 1：UE 不可用时回退到 Three.js

**用户故事：** 作为用户，我希望在 UE 渲染不可用时仍能看到 3D 场景，以便核心功能不受影响。

#### 验收标准

1. 当 UE 实例未启动或连接失败时，系统应在 3 秒内切换到 Three.js 渲染。
2. Three.js 场景应保持与 UE 场景相同的布局和角色位置。
3. 从 Three.js 切回 UE 时应平滑过渡，无闪烁。
4. 降级过程中不应丢失任务状态。

### 需求 2：低带宽降分辨率

**用户故事：** 作为用户，我希望在网络不佳时仍能流畅观看，即使画质有所下降。

#### 验收标准

1. 系统应在检测到带宽不足时自动降低视频流分辨率。
2. 降分辨率应分级进行（1080p → 720p → 480p）。
3. 带宽恢复后应自动提升分辨率。
4. 分辨率变化应在 UI 上有提示。

### 需求 3：移动端预渲染视频

**用户故事：** 作为移动端用户，我希望通过预渲染视频观看场景，以便在低性能设备上也能使用。

#### 验收标准

1. 系统应在移动端自动选择预渲染视频模式。
2. 预渲染视频应覆盖主要任务场景。
3. 视频播放应与任务进度同步。

### 需求 4：降级状态 UI 提示

**用户故事：** 作为用户，我希望知道当前处于哪种渲染模式，以便了解体验差异的原因。

#### 验收标准

1. 系统应在降级时显示当前渲染模式提示。
2. 提示应包含降级原因和恢复建议。
3. 用户应可手动切换渲染模式。
