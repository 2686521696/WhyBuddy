# 现状核查：`web-aigc-node-robot_reply`

## 当前结论

`robot_reply` 已补出最小可运行闭环，当前主仓已具备以下能力：

1. 支持直接接收文本回复内容，输出标准化 `reply` 结构。
2. 支持承接 `dialogue` 节点输出，自动继承 `content / reply / observability` 中的回复、引用、工具摘要与链路标识。
3. 支持复用现有 `messageStore.createMessage` 与 `sessionStore.appendLLMExchange` 底座，把最终回复写入工作流消息链路与会话交换记录。
4. 已补节点级测试与路由级测试，验证 standalone 回复、引用与工具摘要输出、会话落点，以及与 `dialogue` 输出衔接。

## 已落地文件

- `server/routes/node-adapters/robot-reply-node-adapter.ts`
- `server/routes/robot-reply.ts`
- `server/tests/robot-reply-node-adapter.test.ts`
- `server/tests/robot-reply-routes.test.ts`

## 适配边界

- 当前实现走“最终回复投递节点”模式，不额外调用 LLM。
- 会话侧复用的是现有 `appendLLMExchange` 机制，因此会把来源 prompt 与最终 reply 作为一次会话交换写入。
- 当前未扩展更多富媒体回复格式，优先完成文本、引用、工具摘要和会话链路闭环。
