# 需求文档：实时旁白与对话

## 简介

本规格定义 UE5 场景中的实时旁白与角色对话系统。系统需支持 TTS 驱动角色口型同步、多角色轮流发言调度，以及字幕叠加作为降级方案。

## 需求

### 需求 1：TTS 口型同步

**用户故事：** 作为用户，我希望角色说话时嘴型与语音同步，以便获得沉浸式体验。

#### 验收标准

1. 系统应支持将文本通过 TTS 转换为语音并播放。
2. 语音播放时应驱动角色的口型 Morph Target 同步。
3. 口型同步延迟应控制在 100ms 以内。
4. 系统应支持多种 TTS 引擎（Web Speech API / 第三方 TTS）。

### 需求 2：多角色发言调度

**用户故事：** 作为导演系统，我希望多个角色能按顺序轮流发言，以便呈现自然的对话场景。

#### 验收标准

1. 系统应支持发言队列，按顺序调度角色发言。
2. 当前发言角色应有视觉提示（如头顶对话气泡）。
3. 发言切换时应有自然的停顿（0.5-1 秒）。
4. 系统应支持打断当前发言并切换到新发言者。

### 需求 3：字幕降级

**用户故事：** 作为用户，我希望在 TTS 不可用时仍能看到对话内容，以便不影响信息获取。

#### 验收标准

1. 系统应在 TTS 不可用时自动切换到字幕模式。
2. 字幕应显示在视频流底部，支持多行显示。
3. 字幕应标注当前发言角色的名称。
4. 字幕显示时间应与文本长度成正比。
