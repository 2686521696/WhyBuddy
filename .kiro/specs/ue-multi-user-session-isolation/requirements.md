# 需求文档：多用户会话隔离

## 简介

本规格定义多用户同时使用 UE 渲染时的会话隔离机制。系统需为每个用户分配独立的 UE 实例，管理会话生命周期，并在资源不足时提供排队与降级策略。

## 需求

### 需求 1：会话隔离

**用户故事：** 作为用户，我希望我的 UE 场景不受其他用户影响，以便获得独立的体验。

#### 验收标准

1. 每个用户应分配独立的 UE 实例，场景状态互不影响。
2. 用户的角色位置、镜头状态、任务进度应在各自实例中独立维护。
3. 一个用户的操作不应影响其他用户的画面。

### 需求 2：会话生命周期管理

**用户故事：** 作为系统，我希望会话能自动创建和回收，以便高效利用资源。

#### 验收标准

1. 用户连接时应自动创建会话并分配 UE 实例。
2. 用户断开后应在 5 分钟内回收会话和实例。
3. 会话空闲超过 10 分钟应自动暂停实例。
4. 用户重新连接时应恢复之前的会话状态。

### 需求 3：资源池化

**用户故事：** 作为系统，我希望 UE 实例能被池化管理，以便减少启动等待时间。

#### 验收标准

1. 系统应维护 UE 实例池，预热一定数量的空闲实例。
2. 用户请求时应从池中分配实例，分配时间 ≤ 5 秒。
3. 实例池应支持动态扩缩容。
4. 实例池应有健康检查，自动替换异常实例。

### 需求 4：排队与降级

**用户故事：** 作为用户，我希望在实例不足时能排队等待，以便最终获得完整体验。

#### 验收标准

1. 当所有实例被占用时，新用户应进入排队状态。
2. 排队状态应在 UI 上显示预计等待时间。
3. 排队超过 2 分钟应提供降级选项（Three.js 或预渲染视频）。
4. 排队用户应按先到先得顺序分配实例。
