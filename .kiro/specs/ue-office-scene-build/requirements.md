# 需求文档：UE 办公室场景搭建

## 简介

本规格定义在 UE5 中重建 Cube Pets 办公室场景的要求。场景需对标当前 Three.js OfficeRoom 的布局和风格，包括家具、灯光、材质等元素，并建立场景资产规范、命名约定与 LOD 策略，为后续角色系统和镜头系统提供可靠的场景基础。

## 需求

### 需求 1：场景布局还原

**用户故事：** 作为产品团队，我希望 UE5 场景与当前 Three.js 办公室布局一致，以便用户在切换渲染引擎时不会感到陌生。

#### 验收标准

1. UE5 场景应还原 Three.js OfficeRoom 的整体空间尺寸与比例。
2. 场景应包含所有主要家具：办公桌、椅子、书架、白板、沙发、会议桌。
3. 场景布局偏差应控制在 Three.js 版本的 ±5% 以内。
4. 场景应包含墙面、地板、天花板的基础材质。

### 需求 2：灯光与材质系统

**用户故事：** 作为美术团队，我希望 UE5 场景具备合理的灯光和材质，以便呈现高品质的办公室氛围。

#### 验收标准

1. 场景应使用 Lumen 全局光照或等效方案，支持实时光影。
2. 场景应包含主光源（天窗 / 日光）、补光（台灯 / 顶灯）和环境光。
3. 材质应使用 PBR 工作流，包含 BaseColor、Normal、Roughness、Metallic 贴图。
4. 灯光参数应支持通过蓝图变量动态调整。

### 需求 3：资产规范与命名约定

**用户故事：** 作为开发团队，我希望场景资产有统一的命名和组织规范，以便多人协作时不产生混乱。

#### 验收标准

1. 所有资产应遵循 `SM_<类别>_<名称>` / `MI_<名称>` / `T_<名称>_<类型>` 命名规范。
2. 资产目录应按 `Content/CubePets/Environment/Office/` 组织。
3. 每个可交互物体应有独立的 Actor 蓝图，便于后续绑定指令。
4. 场景应提供资产清单文档，列出所有模型、材质和贴图。

### 需求 4：LOD 与性能优化

**用户故事：** 作为系统，我希望场景在不同画质档位下都能流畅运行，以便适配不同性能的开发机。

#### 验收标准

1. 主要家具模型应提供至少 2 级 LOD（高模 + 低模）。
2. 场景总面数应控制在 200 万三角面以内（高画质档位）。
3. 贴图应使用虚拟纹理或 Mipmap 策略，避免显存溢出。
4. 场景应在 GTX 1060 级别显卡上以 30fps 以上运行。
