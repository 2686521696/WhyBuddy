# 需求文档：LLM 导演 Prompt 系统

## 简介

本规格定义 LLM 导演的 Prompt 模板系统，负责将任务阶段转换为分镜脚本，生成角色调度指令和镜头编排指令。系统需与现有 Mission Runtime 集成，在任务执行的关键节点自动触发分镜生成。

## 需求

### 需求 1：分镜脚本生成

**用户故事：** 作为导演系统，我希望根据任务阶段自动生成分镜脚本，以便驱动 UE 场景演出。

#### 验收标准

1. 系统应提供 Prompt 模板，输入任务阶段信息，输出结构化分镜脚本。
2. 分镜脚本应包含：镜头类型、角色动作、持续时间、对白提示。
3. 系统应支持不同任务类型的 Prompt 模板（分析任务、生成任务、协作任务）。
4. 生成的分镜脚本应可直接转换为场景指令序列。

### 需求 2：角色调度指令生成

**用户故事：** 作为导演系统，我希望根据任务参与者自动生成角色调度指令，以便角色行为与任务进度同步。

#### 验收标准

1. 系统应根据任务参与的 Agent 列表生成角色出场与退场指令。
2. 系统应根据 Agent 当前状态生成对应的角色动作指令。
3. 角色调度应考虑场景空间约束（如工位容量、会议室座位数）。

### 需求 3：镜头编排指令生成

**用户故事：** 作为导演系统，我希望根据场景内容自动生成镜头编排指令，以便呈现最佳视觉效果。

#### 验收标准

1. 系统应根据活跃角色数量和位置推荐镜头预设。
2. 系统应在角色对话时自动切换到对话镜头（正反打）。
3. 系统应在任务完成时自动切换到全景庆祝镜头。

### 需求 4：与 Mission Runtime 集成

**用户故事：** 作为系统，我希望导演系统能自动响应任务生命周期事件，以便无需手动触发分镜。

#### 验收标准

1. 系统应监听 Mission Runtime 的阶段变更事件。
2. 系统应在任务开始、执行、评审、完成等阶段自动触发分镜生成。
3. 系统应支持手动覆盖自动生成的分镜。
