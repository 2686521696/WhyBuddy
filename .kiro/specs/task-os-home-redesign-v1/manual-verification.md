# 任务操作系统首页重构方案 v1 手工验证

## 当前验证口径（2026-04-20）

- 当前首页四区骨架已落地，手工验证重点改为确认“主次是否稳定、模块是否归位、主流程是否不用跳页”
- 任务输入与澄清当前仍可能出现在中央浮层，不以“必须纯右栏呈现”作为本轮通过条件
- 断点稳定性和桌面视觉主次仍是本 spec 最主要的未完成项

## 1. 首屏主次验证

1. 打开首页
2. 观察第一眼是否先看到中间任务主线区
3. 验证 3D 场景没有压过任务信息

## 2. 输入与澄清验证

1. 在首页发起一个任务，可通过右侧控制区或当前中央发起浮层进入
2. 验证无需跳页即可提交
3. 若需要澄清，验证澄清问题会留在首页壳体内解决，而不是被迫跳转到其他页面

## 3. 任务主线验证

1. 提交任务后观察中间区
2. 验证 `plan -> execute -> review` 清晰可见
3. 验证当前步骤高亮
4. 验证等待决策时中间区明确显示 waiting 状态

## 4. 控制动作验证

1. 在首页控制区触发 approve / reject / modify
2. 验证动作不再隐藏在日志或其他详情 panel 中

## 5. 底部运行区验证

1. 打开首页共享运行证据容器
2. 验证 `Logs`、`Artifacts`、`Runtime` 三类内容都集中在这里
3. 验证没有关键运行证据散落到其他平级主面板
