# 需求文档：状态同步桥

## 简介

本规格定义 UE5 渲染引擎与前端应用之间的双向状态同步机制。UE 场景的当前状态（角色位置、动作、镜头）需要实时同步到前端 store，前端的任务状态变更也需要映射为 UE 场景动画，形成"前端 ↔ UE"的双向数据桥。

## 需求

### 需求 1：UE → 前端状态推送

**用户故事：** 作为前端应用，我希望实时获取 UE 场景的当前状态，以便 UI 面板能反映场景中正在发生的事情。

#### 验收标准

1. UE 应以 10fps 频率推送场景快照（角色位置、动作状态、当前镜头）。
2. 前端应将接收到的场景状态写入 Zustand store。
3. 状态推送应使用增量更新，只推送变化的字段。
4. 推送延迟应控制在 50ms 以内。

### 需求 2：前端 → UE 状态映射

**用户故事：** 作为系统，我希望前端任务状态变更能自动触发 UE 场景动画，以便场景与任务进度保持同步。

#### 验收标准

1. 任务开始时，系统应触发对应角色进入 work 状态。
2. 任务完成时，系统应触发对应角色播放 celebrate 动画。
3. 任务阻塞时，系统应触发对应角色进入 blocked 状态。
4. 映射规则应通过配置文件定义，支持自定义扩展。

### 需求 3：状态一致性保证

**用户故事：** 作为系统，我希望前端 store 与 UE 场景状态始终一致，以便避免 UI 显示与场景画面不匹配。

#### 验收标准

1. 系统应在连接建立时进行全量状态同步。
2. 系统应在检测到状态不一致时自动触发全量同步。
3. 系统应支持手动触发全量同步（用于调试）。
4. 状态同步应有版本号机制，防止旧状态覆盖新状态。
