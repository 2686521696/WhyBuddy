# 需求文档：宠物角色系统

## 简介

本规格定义 UE5 中 Cube Pets 宠物角色的资产体系、骨骼动画、表情系统与动作状态机。每个宠物角色对应一个 Agent，需支持 idle、walk、work、celebrate、talk、blocked 等动作状态，并通过指令协议接收导演系统的调度。

## 需求

### 需求 1：角色资产与骨骼系统

**用户故事：** 作为美术团队，我希望宠物角色有统一的骨骼结构和动画规范，以便高效制作和复用动画。

#### 验收标准

1. 每个宠物角色应使用统一的骨骼层级结构（Root → Spine → Head / Arms / Legs）。
2. 角色模型面数应控制在 5000 三角面以内（单角色）。
3. 角色应支持至少 4 种外观变体（颜色 / 配饰），通过材质参数切换。
4. 骨骼应支持 IK 约束，用于角色与场景物体的交互动画。

### 需求 2：动作状态机

**用户故事：** 作为导演系统，我希望通过指令切换角色的动作状态，以便编排角色在场景中的行为。

#### 验收标准

1. 角色应支持以下动作状态：idle、walk、work、celebrate、talk、blocked。
2. 状态之间的切换应有过渡动画（blend），过渡时间不超过 0.3 秒。
3. 系统应支持通过指令协议触发状态切换。
4. 角色在无指令时应自动进入 idle 状态并播放待机动画。

### 需求 3：表情系统

**用户故事：** 作为用户，我希望宠物角色有丰富的面部表情，以便感受角色的情绪变化。

#### 验收标准

1. 角色应支持至少 6 种基础表情：neutral、happy、sad、thinking、surprised、angry。
2. 表情应通过 Morph Target / Blend Shape 实现，支持平滑过渡。
3. 表情应可与动作状态组合（如 work + thinking、celebrate + happy）。
4. 系统应支持通过指令协议设置角色表情。

### 需求 4：角色与 Agent 映射

**用户故事：** 作为系统，我希望每个宠物角色与一个 Agent 实体绑定，以便角色行为反映 Agent 的工作状态。

#### 验收标准

1. 系统应维护 agentId → characterId 的映射表。
2. Agent 状态变更（如开始工作、完成任务、遇到阻塞）应自动映射为角色动作状态。
3. 系统应支持动态添加和移除角色（对应 Agent 的上线和下线）。
4. 映射关系应通过配置文件定义，支持热更新。
