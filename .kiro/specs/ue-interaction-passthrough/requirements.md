# 需求文档：交互事件透传

## 简介

本规格定义前端鼠标/触摸事件透传到 UE5 的机制。用户在浏览器中的点击、拖拽等操作需要传递到 UE 场景，实现点击角色触发详情、拖拽旋转视角等交互。系统需处理输入通道配置和 UI 层与场景层的事件冲突。

## 需求

### 需求 1：鼠标事件透传

**用户故事：** 作为用户，我希望在视频流上点击角色时能查看角色详情，以便了解 Agent 的工作状态。

#### 验收标准

1. 系统应将视频流区域的鼠标点击事件透传到 UE5。
2. UE5 应能识别点击位置对应的场景物体（角色 / 家具）。
3. 点击角色时应触发前端角色详情面板。
4. 点击空白区域时不应触发任何操作。

### 需求 2：视角拖拽控制

**用户故事：** 作为用户，我希望通过拖拽旋转 UE 场景视角，以便从不同角度观察办公室。

#### 验收标准

1. 系统应支持鼠标拖拽旋转视角（水平 360°、垂直 -30° 到 60°）。
2. 系统应支持鼠标滚轮缩放（限制最小和最大距离）。
3. 拖拽操作应平滑流畅，延迟不超过 50ms。
4. 拖拽操作不应与 UI 浮层的拖拽交互冲突。

### 需求 3：输入通道管理

**用户故事：** 作为系统，我希望能灵活配置哪些输入事件透传到 UE、哪些被 UI 拦截，以便避免交互冲突。

#### 验收标准

1. 系统应支持配置输入通道模式：全透传、全拦截、智能分发。
2. 智能分发模式下，UI 元素上的事件被拦截，空白区域的事件透传。
3. 系统应支持临时锁定输入通道（如弹窗打开时禁止透传）。
4. 输入通道切换应即时生效，无需重新连接。
