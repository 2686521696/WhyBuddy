# 任务运行可视化收敛方案 v1 手工验证

## 当前验证口径（2026-04-20）

- 当前首页已具备共享运行证据容器，`Logs / Artifacts / Runtime` 以 tabs 形式进入同一个折叠层
- 手工验证重点是“证据是否归口、步骤流是否稳定、重复入口是否继续减少”，而不是要求存在单独一整条底部 dock
- `socket / callback` 目前仍以轻量摘要展示，验证时以状态同步和可见性为主，不以完整事件明细为本轮通过条件

## 1. 计划与步骤流验证

1. 提交一个任务
2. 验证首页中间区出现步骤流
3. 验证每一步有状态和摘要
4. 验证当前步骤高亮

## 2. 日志流验证

1. 打开首页运行证据容器中的 `Logs`
2. 验证日志实时追加
3. 验证暂停滚动后页面不强制跳动
4. 验证错误日志高亮

## 3. Artifact 验证

1. 执行产生 artifact 的任务
2. 打开首页运行证据容器中的 `Artifacts`
3. 验证文件列表出现
4. 验证点击后能预览或下载

## 4. Runtime 验证

1. 打开首页运行证据容器中的 `Runtime`
2. 验证能看到 executor 状态、最近动作、异常信息，以及 `socket / callback` 的轻量状态
3. 验证任务取消、失败或等待时状态同步更新

## 5. Decision 验证

1. 触发一个需要人工决策的任务
2. 验证中间步骤流显示 waiting
3. 验证右侧决策动作与当前步骤联动
