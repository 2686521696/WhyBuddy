# 需求文档：场景指令协议

## 简介

本规格定义 LLM 导演到 UE5 渲染引擎之间的结构化指令协议。协议基于 JSON-RPC over WebSocket，覆盖角色移动、动作触发、镜头切换、场景状态变更与特效触发等指令类型，并提供指令验证、队列管理、超时与错误处理机制。

## 需求

### 需求 1：指令格式与传输通道

**用户故事：** 作为 LLM 导演模块，我希望通过结构化 JSON 指令控制 UE 场景，以便实现自动化分镜编排。

#### 验收标准

1. 系统应定义基于 JSON-RPC 2.0 的指令格式，包含 method、params、id 字段。
2. 指令传输应通过 WebSocket 长连接，支持双向通信。
3. 每条指令应包含唯一 requestId，用于追踪执行结果。
4. 系统应支持批量指令发送（batch request）。

### 需求 2：指令类型覆盖

**用户故事：** 作为导演系统，我希望能控制角色、镜头、场景和特效，以便编排完整的演出流程。

#### 验收标准

1. 系统应支持以下指令类型：角色移动（moveTo）、动作触发（playAnimation）、镜头切换（setCamera）、场景状态变更（setSceneState）、特效触发（playEffect）。
2. 每种指令类型应有明确的参数 schema 定义。
3. 系统应支持指令参数的 JSON Schema 校验，非法参数应返回错误。
4. 系统应支持自定义扩展指令类型。

### 需求 3：指令队列与执行控制

**用户故事：** 作为系统，我希望指令能按顺序可靠执行，以便保证演出时序正确。

#### 验收标准

1. 系统应维护指令执行队列，按接收顺序依次执行。
2. 系统应支持指令优先级，高优先级指令可插队执行。
3. 每条指令应有超时时间（默认 10 秒），超时未完成应返回超时错误。
4. 系统应支持取消排队中的指令。

### 需求 4：错误处理与重试

**用户故事：** 作为调用方，我希望在指令执行失败时获得明确的错误信息，以便进行重试或降级处理。

#### 验收标准

1. 系统应定义标准错误码：INVALID_PARAMS、EXECUTION_FAILED、TIMEOUT、QUEUE_FULL、NOT_CONNECTED。
2. 指令执行失败时应返回包含错误码、错误消息和原始 requestId 的响应。
3. 系统应支持可重试标记，调用方可据此决定是否重试。
4. 连接断开时，队列中未执行的指令应标记为 CANCELLED 并通知调用方。
