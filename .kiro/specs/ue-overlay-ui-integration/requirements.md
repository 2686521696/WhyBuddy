# 需求文档：UI 浮层叠加集成

## 简介

本规格定义在 UE5 视频流上叠加半透明 UI 元素的方案。前端 React UI（侧边栏、任务面板、发起面板等）需要渲染在视频流之上，同时支持鼠标事件的穿透与拦截策略，以及 HUD 元素（状态指示器、进度条）在视频流上的渲染。

## 需求

### 需求 1：UI 浮层分层渲染

**用户故事：** 作为用户，我希望在观看 UE 渲染画面的同时能操作侧边栏和任务面板，以便不中断工作流。

#### 验收标准

1. 系统应将 React UI 渲染在视频流 `<video>` 元素之上，使用 CSS z-index 分层。
2. UI 浮层应支持半透明背景（backdrop-filter: blur），不完全遮挡视频画面。
3. 侧边栏、任务面板、发起面板应保持现有布局和交互行为不变。
4. UI 浮层的渲染不应影响视频流的帧率。

### 需求 2：鼠标事件穿透与拦截

**用户故事：** 作为用户，我希望在 UI 空白区域点击时能与 UE 场景交互，在 UI 元素上点击时能操作 UI，以便两种交互模式无缝切换。

#### 验收标准

1. UI 浮层的空白区域应将鼠标事件穿透到底层视频流（pointer-events: none）。
2. UI 元素（按钮、输入框、面板）应拦截鼠标事件（pointer-events: auto）。
3. 系统应支持配置哪些区域穿透、哪些区域拦截。
4. 拖拽操作（如视角旋转）应在 UI 空白区域正常工作。

### 需求 3：HUD 元素渲染

**用户故事：** 作为用户，我希望在视频流上看到角色状态指示器和任务进度条，以便快速了解当前状态。

#### 验收标准

1. 系统应支持在视频流上叠加 HUD 元素：角色名称标签、状态图标、进度条。
2. HUD 元素应跟随 UE 场景中角色的屏幕坐标定位。
3. HUD 元素应在角色被遮挡或离开视野时自动隐藏。
4. HUD 元素的更新频率应与视频流帧率同步。

### 需求 4：响应式布局适配

**用户故事：** 作为用户，我希望在不同屏幕尺寸下 UI 浮层和视频流都能正确显示，以便在不同设备上使用。

#### 验收标准

1. 视频流应自适应容器尺寸，保持 16:9 宽高比。
2. UI 浮层应在桌面端（≥1280px）显示完整侧边栏，在窄屏（<1280px）折叠为图标。
3. HUD 元素应根据视频流缩放比例调整位置和大小。
